# Decision Log — 왜 일지 형태로 출시하는가

> **이 문서는 트레인오라클의 첫 출시 버전이 왜 "AI 코칭 도구" 가 아니라 "일지" 인지를 설명합니다.**
> 모든 결정은 호준 코치 (장호준) 와의 두 번의 디자인 세션 (2026·06) 에서 나왔습니다.
> 새로 합류하는 사람은 **이 문서를 가장 먼저** 읽어주세요.

---

## 한 줄로

> **"이 도구는 매일 들여다보고 싶은, 손으로 쓰던 일지여야 한다.**
> **AI 는 가끔 옆에서 패턴을 짚어주는 정도. 메인은 사용자가 쌓는 데이터."**

이 한 문장이 v3 의 모든 디자인 결정을 지배합니다. 의심스러우면 이 문장에 비추어 판단하세요.

---

## 어떻게 여기 도착했나 — 두 번의 피벗

### v1 (디자인 스프린트 초기)
- "AI 코칭 플랫폼" — 둥근 모서리, Instrument Serif, 어두운 테마, 그라데이션
- 톤: SaaS 스타트업 흔한 미감
- **버린 이유**: 호준 코치의 30년 방법론은 "느낌" 이 아니라 "측정과 기록". SaaS 톤이 그 무게를 못 받쳐줬음. 또 의료·과학 도구의 무게를 원했음.

### v2 (Scientific Minimalism — Tufte × Linear)
- 직각, hairline 1px, 모노 + sans 페어, 색은 정보로만 (장식 X)
- AI 가 화면 중심 — Dashboard / SessionDetail / Calendar / **AIChat / Inbox**
- **5개 데스크탑 화면 완성**
- **버린 이유**: 만들고 보니 **"AI 가 코치를 대체한다"** 는 그림이 됨. 호준 코치가 원한 건 그게 아니었음. 코치는 옆에 있고, **선수 본인이 일지를 쌓는 행위 자체** 가 핵심인데 그 행위가 안 보였음.

### v3 (Journal-first, mobile) — **현재 출시 버전**
- 메인 = **오늘 일지 쓰기 CTA + 어제 요약**
- AI = 1-2 줄 인사이트, 사용자가 끌 수 있는 토글
- 모바일 우선 (선수는 훈련 직후 / 잠들기 전 / 경기장에서 폰을 쓴다)
- 두 미감 동시 — **UI 영역은 v2 직각 hairline 그대로, 사용자 입력 영역만 손글씨 + 종이톤**

## v3 피벗의 핵심 단서 — 14개 질문에서 나온 원문

두 번째 세션 시작 때 14개 질문을 했고, 호준 코치의 답변에서 다음 단서들이 나왔습니다. **이건 디자이너의 해석이 아니라 호준 코치의 말 그대로 옮긴 것** 입니다.

### 1. 메인 화면이 뭐여야 하나?
> **"오늘 일지 쓰기 CTA + 어제 요약."**
> AI 가 뭘 추천하는 게 아니라, **선수가 오늘 뭘 쓸지 정하는 게 먼저**.

→ Home 의 1픽셀 1픽셀이 "쓰러 가기" 로 향하게 설계.

### 2. 누가, 언제 쓰는가?
> **4 진입점:**
> 1. **훈련 직후** — 운동 끝나고 땀 닦으면서, 폰으로. 빠르게.
> 2. **하루 마무리** — 자기 전 침대에서. 그날 컨디션·기분·통증 종합.
> 3. **경기 직전** — 경기장 가는 길에. 목표 페이스·전략 적어두기.
> 4. **경기 직후** — 결과 + 회복 + "다음엔" 메모.

→ `LogEntry.jsx` 의 4 폼 분기 (post-session / evening / race-pre / race-post) 가 이 4 진입점에서 그대로 나왔습니다. 하나의 거대한 폼이 아니라 **상황별 폼 4개** 인 이유.

### 3. 어떤 일지를 원하나?
> **"손으로 적던 일지처럼."**
> **"꾸미고 싶다."**
> **"프린트해서 가지고 싶다."**

→ 이 세 문장이 v3 의 미감 전체를 결정.
- "손으로 적던" → `Caveat` (영문) + `Gowun Dodum` (한글) + `--ink-blue` (만년필 청색)
- "꾸미고 싶다" → 마스킹 테이프 (`.tape`), 잉크 스탬프 (`.stamp`), 형광펜 (`.hl`), 도서관 카탈로그 카드 헤더 (`.idx-card`)
- "프린트해서" → `@media print` A5 portrait 인쇄 스타일, 모바일 베젤 자동 해제

### 4. AI 는 어디에 있나?
> **"AI 거의 안 보임. Home 에 1-2줄. 끄고 켜기."**

→ Tweaks 패널의 "AI 표시" 토글로 노출. 기본은 켜져 있지만 **선수가 거슬리면 끌 수 있어야** 함. v2 의 `AIChat.jsx` / `Inbox.jsx` 는 v3 에 흡수하지 않고 **코치 페르소나용 별도 모듈로 archive**.

### 5. 변형은 몇 개?
> **"화면별 2-3개 변형. Tweaks 로 전환."**

→ Home A/B/C, LogDetail A/B, Trends A/B. 각각 다른 사용자를 노림 (→ `VARIANTS.md` 참고).

### 6. 데이터는 누가 입력?
> **"센서 자동 + 손으로 적는 메모. 둘 다 한 자리에."**

→ 한 IndexCard 안에 자동 메트릭 (avg HR, pace, splits) + 손글씨 메모 (`.hand` / `.hand-pencil`) 가 공존. 이게 다른 앱과 가장 다른 부분.

### 7. 통증·부상은?
> **"숨기지 말고 적어두고, 며칠 째인지 보여줘. 안 그러면 부상 키운다."**

→ Pain scale 5단계 (`--pain-1..5`) + body diagram 마커 + "3d 째" 라는 경과 표기. `BodyDiagram` 컴포넌트에서 클릭으로 부위 지정.

### 8. 기분은?
> **"수치보다 색깔로. 5단계면 충분."**

→ `MoodStrip` — 5단계 bar (cool slate → warm clay red). 숫자 표시 X.

### 9. 에너지 시스템 코드 (BA/LT/V2/GL/AP/RE) 는?
> **"내가 30년 가르친 시스템이다. 절대 색으로만 인코딩하지 말고, 코드 + 도트 + 텍스트 다 같이 보이게."**

→ v2 규칙 그대로 유지. 색만으로 구분하면 색맹 사용자가 못 읽고, 인쇄했을 때 흑백이면 사라짐.

### 10. 코치는 어디 있나?
> **"코치는 일지를 읽는 사람이지, 시스템 안에 사는 캐릭터가 아니다."**

→ v2 의 "AI 코치" 페르소나는 v3 출시판에선 빠짐. 추후 코치 전용 뷰는 별도 모듈.

### 11-14. (시각 톤·폰트·언어·인쇄)
- **시각 톤**: "연구자 노트북 + 모눈 일지장"
- **폰트**: 손글씨 두 세트 + 기존 v2 의 Inter / JetBrains Mono 그대로
- **언어**: 한글 우선, 모노 라벨만 영문 약어 허용
- **인쇄**: A5, 한 일지 한 페이지, 베젤·status bar 제거

## 첫 출시가 "일지" 인 이유 — 사업적 근거

> 이 부분은 호준 코치의 의사결정이고, 디자이너로서 받아쓴 내용입니다.

1. **차별화** — Strava (기록 자랑), TrainingPeaks (분석) 사이에 **"매일 나를 쓰는 도구"** 라는 빈 자리가 있다. AI 코칭 도구는 이미 레드오션 진입 중이라 미루는 게 낫다.
2. **데이터 축적이 먼저** — AI 코칭이 진짜 의미 있으려면 선수 한 명당 최소 6개월의 일지 데이터가 필요. 일지를 먼저 만들고 사람들이 쓰게 하는 게 순서.
3. **호준 코치 본인의 운영 도구** — 호준 코치가 본인이 가르치는 선수들에게 매일 쓰게 할 수 있는 형태. 1차 사용자가 명확.
4. **"꾸미고 싶다" 는 retention 의 단서** — 사용자가 자기 일지를 예쁘게 꾸미기 시작하면 그건 anchoring effect. SaaS 트레이닝 앱은 이 감정적 lock-in 이 없다.

## v3 가 깨면 안 되는 v2 규칙

피벗했다고 모든 걸 바꾼 게 아닙니다. **UI 텍스트·시스템 영역은 v2 그대로** 입니다.

- 직각 (`border-radius ≤ 4px`). avatar circle (50%) 만 예외.
- 그라데이션 금지.
- **UI 텍스트** 에서 이모지 금지. 사용자 입력 (메모/일기) 만 자유.
- 모든 숫자 `tabular-nums`.
- 에너지 시스템 인코딩 = 도트 + 코드 + underline. 배경색으로 인코딩 X.

손글씨와 종이톤은 **사용자가 입력한 영역에만** 들어갑니다. 시스템 라벨·버튼·meta 는 모노 + sans 그대로.
