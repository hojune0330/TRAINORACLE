# Formation Composite Session And Load Evidence Review

## 결론부터

복합 훈련은 하나의 “피로도 색”으로 뭉개면 안 된다. 세션은 선수에게는 한 일정으로
보이되, 분석에서는 달리기·웨이트·플라이오·언덕·대체 유산소 구성요소와 순서·간격을
보존한다. 전체 세션의 sRPE를 구성요소에 비율로 나누는 근거는 없으며, 단위가 다른 값을
합쳐 하나의 준비도·안전 점수로 만들 수 없다.

## 복합 세션 근거

청소년·성인 주자 연구는 웨이트와 플라이오가 경제성·근력·파워·일부 수행에 도움이 될 수
있음을 보여준다. 그러나 구성요소의 효과가 곧 9.5일 안의 특정 배치 효과는 아니다.

- `heavy resistance -> matched plyometric`과 3-4분 회복은 좁은 프로토콜에서만
  `CONDITIONALLY_SUPPORTED`다. `SRC-PMID-41312139`, `FRV2-C-007`.
- 청소년 중거리에서 달리기와 웨이트의 보편 순서, 같은 날 몇 시간 간격, 다음 날 배치는
  모두 `UNKNOWN`이다.
- 성인 자료는 폭발력·최대근력·유산소 결과에 따라 방향이 다르며 보편적인
  `strength first` 규칙을 지지하지 않는다. 관련 출처는 claim matrix의
  `FRV2-CLAIM-C-003`에 정본 ID로 연결되어 있다.
- 자전거와 대체 유산소는 달리기와 같은 부하 또는 보장된 회복 훈련으로 취급할 수 없다.

## 구성요소 표현

```yaml
session:
  session_id: required
  planned_at: required
  actual_at: optional
  components:
    - component_type: RUN | HILL | STRENGTH | PLYOMETRIC | ALT_AEROBIC | OTHER
      goal: required
      modality: required
      dose: structured
      order_index: required
      separation_minutes: explicit_or_unknown
      relation: SAME_SESSION | SAME_DAY | ADJACENT_DAY
      provenance: required
```

알 수 없는 순서·간격·용량은 `UNKNOWN`으로 남긴다. 추정해서 채우지 않는다.

## 부하 데이터 계약

| 계열 | 예 | 규칙 |
|---|---|---|
| 외적 부하 | 시간·거리·고도·속도·반복·파워 | 물리 단위와 측정 범위를 보존 |
| 내적 부하 | 심박·RPE·세션 RPE | 척도·앵커·질문 시점 보존 |
| 파생값 | TRIMP 계열·계산값 | 공식 ID·버전·원천 ID 필수 |
| 맥락 | 수면 우려·대회·학교·날씨·표면 | 부하와 합산하지 않음 |

기기값은 모델·착용 위치·알고리즘/펌웨어·측정 시각을 가능한 범위에서 남긴다. 방법이
바뀌면 이전 시계열과 자동 연결하지 않는다. 누락은 0이 아니다.

## 합산과 중복 방지

1. 같은 물리량·단위·시간/공간 범위·출처 의미가 맞을 때만 더한다.
2. 부모 세션과 세부 구성요소가 같은 관측을 가리키면 둘 중 하나만 계산한다.
3. 전체 세션 sRPE는 부모에 둔다. 독립적으로 측정한 구성요소 RPE가 없으면 나누지 않는다.
4. TRIMP 계열은 공식이 다르면 서로 같은 값처럼 비교하거나 합치지 않는다.
5. ACWR은 설명용 실험값조차 기본 비활성이다. 안전/위험 구간·행동·처방 권한이 없다.

## 사용자에게 보이는 방식

선수의 첫 화면은 지나치게 세분화하지 않는다. 하루 타일에는 **주된 목적과 전체 체감
부하**를 먼저 보여주고, 탭하면 구성요소·순서·간격을 펼친다. 색은 운동 종목이 아니라
첫눈에 이해해야 하는 강도/부담 수준을 기본 축으로 사용하고, 종목은 아이콘·짧은 라벨로
보조한다. 색만으로 의미를 전달하지 않는다.

복합 세션 입력은 “웨이트 + 플라이오 + 달리기”를 한 번에 고를 수 있어야 하며, 각 항목을
다시 입력하게 해서는 안 된다. 전문가는 상세 단위를 볼 수 있고, 중학생 선수는
`오늘의 중심 / 함께 한 운동 / 힘들었던 정도`부터 입력한다.

## 남은 결정

- 구성요소 ID·필수 단위·파생 공식 등록표는 Load Components 결정 패킷에서 승인한다.
- 순서·간격의 직접 청소년 근거가 없으므로 코치 규칙은 버전·근거·예외를 명시한다.
- 앱의 단일 피로 색을 분석값으로 사용하면 안 된다. 표시 요약과 분석 원장을 분리한다.
- 런타임과 자동 계획 변경 권한은 아직 없다.

## 2026-07-17 청소년 1차 자료 보강

청소년·주니어 중거리 주자 직접 연구를 다시 추적했지만, 달리기와 웨이트의 양방향 순서,
같은 세션·같은 날·다음 날 간격, 다음 MAIN까지의 회복을 모두 비교한 시험은 없었다.
확인된 직접 자료는 다음처럼 좁게만 쓸 수 있다.

- `heavy resistance -> matched plyometric`와 3-4분 간격.
- 한 strength session 안의 `plyometric/sprint -> resistance` 순서.
- 일부 endurance volume을 explosive work로 **대체**한 사례.
- 6회 depth jump 뒤 10분 휴식의 급성 warm-up 효과.

서로 다른 목적·용량·기간을 하나의 보편 순서로 합치지 않는다. `추가`와 `대체`를 구분하고,
대체 유산소도 목적·용량·출처가 있는 별도 부하로 저장한다. 쉬운 선수 문구는 “오늘은
달리기와 웨이트를 같이 해요. 순서와 쉬는 시간은 따로 보여줄게요”이며, 72시간이 지났다는
이유로 회복 완료를 표시하지 않는다.
