# Formation Youth Transfer And Participant-Visible Pilot Review

## 결론부터

첫 시험은 9.5일 자동 처방의 효과나 안전성을 증명하는 시험이 아니다. 선수와 코치가 후보를
보고 이해하는지, 누락·충돌·중단이 제대로 드러나는지, 코치에게 인계되는지를 확인하는
**참여자에게 보이는 병렬 비교**다. 선수 몰래 돌아가는 “그림자”가 아니다.

## 시험 흐름

1. 코치가 기존 방식으로 9.5일 계획을 먼저 작성하고 시각·버전을 고정한다.
2. 시스템이 같은 시점의 허용된 입력으로 자동 후보 또는 `NO_CANDIDATE`를 만든다.
3. 선수와 코치 모두 후보·사용된 데이터·사용하지 않은 데이터·불확실성을 본다.
4. 코치는 `동의`, `수정`, `후보 없음이 맞음`, `정보 부족`과 이유를 기록한다.
5. 선수는 쉬운 말로 “왜 나왔는지, 누가 최종 결정하는지”를 다시 설명해 본다.
6. 시간·중단·재개·누락·철회·기술 문제·인계 완료 여부를 기록한다.
7. 생성 후보는 실제 계획을 바꾸지 않는다. 실행은 고정된 코치 계획을 따른다.

## 측정할 것과 측정하지 않을 것

| 측정 | 의미 |
|---|---|
| 후보/무후보/기술 실패/인계 상태 | 시스템 절차가 구분되는지 |
| 코치의 필드별 동의·수정 | 계획 차이가 어디서 나는지 |
| 선수의 설명 이해 | 데이터와 권한을 이해하는지 |
| 입력·비교·설명 완료율 | 절차가 실행 가능한지 |
| 시간·중단·지원 요청 | 부담과 마찰이 어느 정도인지 |
| 철회·삭제·인계 완료 | 권리와 운영이 실제로 작동하는지 |

경기 결과, 부상 수, 기록 향상, 코치 동의율은 첫 시험에서 효능·안전·정답률이 아니다.
단일 선수 누적 기록으로 인과를 말하지 않는다. `FRV2-F-001..016`.

## 청소년 안전 경계

계획 생성에 필요한 건강·참여 질문은 다음 네 상태를 선수에게 그대로 보여준다.

```text
NO_CONCERN_REPORTED | CONCERN_REPORTED | PREFER_NOT_TO_ANSWER | NOT_ASKED
```

`NO_CONCERN_REPORTED`만 그 질문의 응답 완료를 뜻하며, 안전·회복·준비 완료를 뜻하지 않는다.
나머지 상태와 출처 충돌·오래된 제한 기록은 후보 생성을 로컬에서 멈춘다.
`PREFER_NOT_TO_ANSWER`와 `NOT_ASKED`는 우려로 해석하지 않는
`SUPPRESS_ONLY_NO_OUTBOUND_DISCLOSURE`다. 누구에게도 자동으로 보내거나 알리지 않는다.

- 일정 사실은 코치가 확인할 수 있지만 건강 우려를 해제할 수 없다.
- 건강 우려에는 청소년 스포츠의학 지원 경로를 로컬에서 보여줄 수 있다. 비응급 외부 전송은
  정확한 수신자·필드·목적·기간을 보여준 뒤 선수의 별도 명시 선택이 있어야 한다.
- 참여·동의·철회는 선수와 필요한 보호자의 권한이다.
- safeguarding 내용은 훈련 생성기에 전달하지 않는다. 승인된 담당자 경로도 실제로 구성된 경우에만
  선택지로 표시하며, `NOT_CONFIGURED`를 접수나 전달 완료처럼 보여주지 않는다.
- 응급 안내는 지역·연령별 승인 문구가 있을 때만 표시하며 진단·점수화를 거치지 않는다.

## 개인 메모와 공유

`나만의 메모`는 읽기·토큰화·검색색인·요약·분류·안전 신호·코치 알림·계획 입력을 모두
금지한다. 구조화된 우려 입력은 별도 화면이다. 개인 메모에 적었다는 사실이나 적지 않았다는
사실, 길이·작성 시각·빈도·누락도 분석·계획·보상·telemetry·안전 신호로 쓰지 않는다.

선수는 자기 데이터와 메모를 로컬 전체 백업으로 내보낼 수 있어야 한다. 코치·친구 등에게
보여주는 공유는 선수의 명시적 선택으로 두되, **무엇을 누구에게 얼마나 오래 보여주는지**를
매번 확인하고 취소할 수 있어야 한다. 개인 메모는 기본 제외이며 선수가 직접 포함할 때만
공유한다. 이는 `USER_DIRECTED_FILE_OPERATION`이며 분석 동의나 상시 코치 접근이 아니다.
preview·백업·공유는 analytics·보상·Formation 이벤트를 만들지 않는다. 수신자 기기에 내려받은
파일은 앱에서 회수하지 못할 수 있음을 확인 전에 알려야 한다. 공유 선택이 계획 품질이나 보상에
불이익을 주면 안 된다.

## 사용자 경험 요구

- 시작 전에 “자동 후보는 실제 계획을 바꾸지 않는다”를 한 문장으로 알린다.
- 9.5일 진행은 체크·스티커처럼 볼 수 있지만 수행 압박이나 건강 보상으로 만들지 않는다.
- 누락·우려·철회 상태에서 연속 기록 보상을 잃게 하지 않는다.
- 중학생 수준 기본 설명과 코치용 상세 설명은 같은 사실을 다르게 표현해야 한다.
- 선수는 언제든 `잠시 멈춤`, `사람에게 물어보기`, `시험에서 나가기`를 찾을 수 있어야 한다.

모든 중단 화면은 `무슨 일이 생겼는지`, `그대로 유지되는 것`, `지금 할 수 있는 일`,
`누가 해결할 수 있는지`를 쉬운 말로 보여준다. 연결된 코치·검토자가 없거나 오프라인이면
`NO_CONFIGURED_REVIEWER_KEEP_CURRENT_PLAN`으로 현재 코치 작성 계획과 기본 일지를 유지하고,
기록 정정·로컬 백업·나중에 다시 보기·사용자가 직접 공유하기를 제공한다. 더 민감한 내용을
입력해야만 화면을 벗어날 수 있는 막다른 길을 만들지 않는다.

## 외부 차단

청소년 스포츠과학, 스포츠의학/safeguarding, 개인정보, 지역 법·윤리, 제품 책임자 승인이
필요하다. 대한민국의 미성년 참여·동의·데이터 처리 문구는 별도 현지 검토 없이는 사용할 수
없다. 현재 실제 선수 모집·운영·자동 처방 활성화는 금지되어 있다.
