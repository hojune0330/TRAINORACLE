1. **코덱스**: 토큰이 많이 드는 대량 작업 전담 + 사장님이 결정을 내리는 창구.
2. **페이블(Fable)**: 고등 판단·`app/` 소유·독립 검수. 사장님과 직접 협의.
3. **PR 합의 원칙**: 모든 PR은 사장님·페이블·코덱스 **3자가 함께 결정**한 뒤
   병합한다. PR이 협업의 기준 단위다.
4. 기존 불변식 유지: 검수자는 지적만 하고 직접 수정하지 않는다(rule 14).
   최신 명시적 책임자 결정이 우선한다. `runtime_authority: false`.

**코덱스가 할 일**: 아래 안건을 사장님께 순서대로 제시하고, 결정마다

문서(WO012 등)는 개별로 받는다.

## 4. 코덱스 제출물 형식 (PR 규칙)

1. 결정 세트마다 PR 1개. 제목: `docs(decision): <안건> 책임자 결정 기록`.
2. PR 본문에 ①사장님 응답 verbatim ②해석·정의 ③반영 파일 ④검증기 결과
   ⑤이 결정이 열지 **않는** 것(권한 경계)을 명시.
3. **3자 합의 규칙**: 코덱스 제출 → 페이블 독립 검수(지적만) → 사장님 최종
   병합. 셋 중 하나라도 이의가 있으면 병합하지 않는다.
4. 검증기 갱신을 상태 변경과 같은 PR에 포함 (분리 시 CI 깨짐).
5. `owner_decision`/전문가 결정란을 실제 서명 없이 격상하는 변경은 금지 —
   기존 fail-closed 구조 유지.
