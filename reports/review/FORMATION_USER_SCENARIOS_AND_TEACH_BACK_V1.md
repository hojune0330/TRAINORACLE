## 목적

선수·코치·보호자가 9.5일 기본안의 **선택**, **검토**, **실행 확정**, **자동안 중단**을
구분하는지 확인한다. 설명을 읽었다는 체크가 아니라 자기 말로 다시 설명하는
teach-back을 사용한다. 정답을 유도하거나 틀렸다고 벌점을 주지 않으며, 이해하지 못하면
문구와 구조를 고친 뒤 다시 본다.

## 공통 화면 원칙

1. 결과를 첫 문장에 둔다: 오늘 할 일, 다음 대회, 지금 실행되는 계획.
2. 시스템·코치·선수 중 누가 무엇을 했는지 주어를 숨기지 않는다.
3. 기본 설명은 짧게 두고 자세한 근거는 펼쳐본다.
4. 자주 보는 일정 전환은 즉시 반응하고 장식 애니메이션을 쓰지 않는다.
5. 상태를 색·아이콘·스티커만으로 전달하지 않는다.
6. 확인 버튼을 누르기 전 범위·효과·되돌리기를 같은 화면에서 본다.

## 사용자군별 검토

### 중학생 선수

- 최소 5명, 읽기 수준과 운동 경험을 다양하게 모집한다.
- U-01·04·05·07·09·10을 자기 말로 설명하게 한다.
- 한 번의 설명 뒤 핵심 질문 6개 중 5개 이상을 맞히고, 틀린 항목이 개인정보·실행
  권한·안전 허가라면 점수와 무관하게 문구를 고친다.
- “모르겠어요”와 질문하기는 실패나 보상 손실로 처리하지 않는다.

### 코치

- 최소 3명에게 U-01~09의 현재 계획·기본안·확정안 lineage를 찾게 한다.
- 다중 대회·누락 MAIN·template 부재에서 새 예외 계획을 발명하지 않는지 확인한다.
- 5분 안에 오늘 실행 계획과 자동안 중단 이유를 찾아야 한다.

### 보호자와 개인정보 검토자

- 공유가 상시 코치 접근이나 분석 동의로 바뀌지 않음을 확인한다.
- 수신자·필드·목적·기간·preview·철회가 한 흐름에서 보이는지 검토한다.
- 청소년 거절이 shadow·공유를 멈추고 기본 일지와 보상을 보존하는지 확인한다.

### 보조기술 사용자

- NVDA+Chrome 또는 승인된 동등 조합, 키보드 전용, 200% 및 400% equivalent reflow,
  고대비, grayscale/색각 조건, reduced motion, 320px·375px 모바일을 검토한다.
- 초점 순서는 결과 -> 현재 실행 계획 -> 바뀐 점 -> 다음 행동이어야 한다.
- 긴 한글 라벨이 잘리거나 버튼·상태와 겹치면 실패다.
- 상태 변화는 live region으로 짧게 알리고 전체 화면을 반복 낭독하지 않는다.

## 보상과 진행 표시

체크·스티커는 훈련량이나 속도를 밀어붙이지 않는다. 다음 행동을 같은 수준으로 인정한다.

- 훈련 기록.
- 계획된 휴식 기록.
- 통증·질병·강한 피로의 정직한 보고.
- 잘못된 기록 정정.
- 코치 확인이 필요한 상태에서 무리하지 않고 대기.

shadow 참여, 메모 공유, 무통증 응답, 완주, 빠른 기록에는 보상을 묶지 않는다. 참여 중단과
공유 철회도 연속 기록을 깨지 않는다.

U-07의 실제 시작·종료는 사용자가 또는 명시적으로 승인된 코치가 간격 설명에 필요한
경우에만 구조화해서 기록한다. 앱은 GPS·위치 이력·원문 메모로 이를 추정하지 않는다.
