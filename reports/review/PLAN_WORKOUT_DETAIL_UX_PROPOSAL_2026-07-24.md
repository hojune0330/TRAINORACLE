## 0. 요약 (한 문단)

지금 계획 베타는 세션을 "편안한 지구력 / 조절된 자극 / 휴식" 3종으로만 보여준다.
이 기획은 그 자리에 **훈련 라이브러리 12종**(크루즈 인터벌, VO2max 인터벌,
해당계 부하 인터벌, 복합 인터벌, 템포런, ATP 스피드 계열 등)을 얹고, 각 세션을
`10×400m ×2세트 @5K RP · r70″ · R10′` 같은 **한 줄 숫자 표기**로 제시한다.
어려워 보이지 않도록 **3층 정보 구조**(쉬운 이름 → 숫자 한 줄 → 접힌 설명)를
쓰고, 페이스 숫자는 **사용자가 직접 입력한 기록이 있을 때만** 계산한다(추정·
날조 금지 — 기존 안전 원칙 유지).

## 1. 왜 지금인가

- 일지 쪽에는 이미 에너지 시스템 6분류(BA/LT/V2/GL/AP/RE)와 색상 체계가
  살아 있다 (`PostSessionForm.ENERGY_SYSTEMS`, `LogDetail.SYSTEM_META`).
  계획 쪽만 3종 뭉뚱그림이라 **기록 언어와 계획 언어가 어긋난다.**
- "조절된 자극 25~40분 RPE 5~6"은 안전하지만 심심하다. 훈련을 아는 사용자는
  물성이 없다고 느끼고, 초보는 뭘 해야 할지 모른다. 둘 다 이탈 요인.
- 숫자 표기(`10×400m r70″`)는 육상부 화이트보드 언어다. 이 언어를 쓰되
  **처음 보는 사람도 3초 안에 해독**되게 만드는 것이 이 기획의 전부다.

## 2. 훈련 라이브러리 12종 (에너지 시스템 매핑)

기존 6색 시스템 배지를 그대로 재사용한다. 새 색·새 분류 없음.

| # | 훈련 이름 (쉬운 이름) | 시스템 배지 | 몸에서 하는 일 (카드 1문장) | 표기 예 |
|---|---|---|---|---|
| 1 | 이지런 (편안한 달리기) | BA 파랑 | 심장과 다리의 기초 배터리를 키워요 | `40′ @이지 · RPE 3~4` |
| 2 | 롱런 (길게 달리기) | BA 파랑 | 오래 버티는 연료 탱크를 넓혀요 | `70~90′ @이지` |
| 3 | 템포런 | LT 노랑 | "힘들지만 유지 가능한" 경계를 끌어올려요 | `20′ @T페이스 · RPE 6~7` |
| 4 | 크루즈 인터벌 | LT 노랑 | 템포런을 쪼개서 더 좋은 자세로 채워요 | `4×1600m @T · r60″` |
| 5 | VO2max 인터벌 | V2 주황 | 산소를 쓰는 최대 엔진 출력을 올려요 | `5×1000m @5K RP · r2′30″` |
| 6 | 짧은 VO2 인터벌 | V2 주황 | 엔진 출력 훈련을 짧게 잘라 부담을 줄여요 | `10×400m ×2set @5K RP · r70″ · R10′` |
| 7 | 해당계 부하 인터벌 | GL 빨강 | 젖산이 확 오르는 구간을 견디는 힘을 만들어요 | `3×300m @1500m RP · r5′` |
| 8 | 복합 인터벌 | GL 빨강 | 서로 다른 속도를 한 세션에 섞어 경기 감각을 만들어요 | `600m@3K RP + 4×200m@1500m RP · R8′` |
| 9 | ATP 스피드 — 스타트 | AP 보라 | 출발 반응과 첫 3걸음의 폭발력을 깨워요 | `6×스타트 대시 20m · r걷기 복귀` |
| 10 | ATP 스피드 — 단거리 대시 | AP 보라 | 최고 속도 스위치를 안전하게 켜요 | `4×30m + 4×50m · r2′~3′ 완전 회복` |
| 11 | ATP 스피드 — 언덕 스프린트 | AP 보라 | 넘어질 걱정 없이 폭발력을 기르는 가장 안전한 방법 | `6×3″ 언덕 스프린트 · r걸어 내려오기` |
| 12 | 회복 조깅 / 휴식 | RE 회색 | 강한 날을 소화시키는 날이에요 | `20~30′ 아주 편하게` 또는 `완전 휴식` |

경험 밴드별 노출 규칙:
- `NEW_TO_RUNNING`: 1·2·3·12만. 인터벌 계열은 후보에 넣지 않는다.
- `DEVELOPING`: + 4·5·6·11 (해당계·복합·플랫 대시 제외).
- `EXPERIENCED`: 전체 12종.
- 보수형(CONSERVATIVE) 후보는 밴드와 무관하게 GL/AP 계열 제외 유지.

## 3. 숫자 표기법 (notation) 규격

**한 줄 문법**: `[반복]×[거리|시간] [×세트] @[강도 앵커] · r[반복 간 휴식] · R[세트 간 휴식]`

| 토큰 | 뜻 | 예 |
|---|---|---|
| `10×400m` | 400m를 10번 | |
| `×2set` | 그 묶음을 2세트 | |
| `@5K RP` | 5000m 레이스 페이스로 | RP = Race Pace |
| `@T` | 템포(역치) 페이스로 | |
| `@이지` | 대화 가능한 편한 속도로 | |
| `r70″` | 반복 사이 휴식 70초 | 소문자 r = rep 휴식 |
| `R10′` | 세트 사이 휴식 10분 | 대문자 R = set 휴식 |
| `′ ″` | 분 · 초 | 60′=60분, 70″=70초 |

**해독 가능성 장치 (이 기획의 핵심 UX):**
1. **모든 토큰은 탭 가능** — 기존 `TermHelp` 컴포넌트 재사용. `r70″`을 탭하면
   "반복 사이에 70초 쉬어요. 걷거나 제자리에 서 있어도 돼요." 바텀시트.
2. **첫 만남 1회 안내** — 표기를 처음 보는 세션 카드에만 밑줄 점선 + "탭하면
   풀이" 캡션 1줄. 두 번째부터는 캡션 제거(로컬 플래그).
3. **음성처럼 읽어주는 풀이 줄** — 카드 접기 안에 항상 한국어 완문 병기:
   "400미터를 5K 목표 페이스로 10번, 사이마다 70초 휴식. 10분 쉬고 한 세트 더."
   스크린리더는 기호 줄 대신 이 완문을 읽는다(aria-label).

## 4. 페이스 앵커(@RP) 안전 규칙 — 날조 금지

숫자 페이스는 **신뢰 사다리** 3단계로만 제공한다. 상위 단계가 없으면 아래로.

| 단계 | 조건 | 세션 카드에 보이는 것 |
|---|---|---|
| A. 실측 기록 | 사용자가 경기 일지(race) 또는 PB를 **직접 입력** | `@5K RP ≈ 4:10/km → 400m당 1′40″` 환산까지 표시 |
| B. 목표 페이스 | 사용자가 목표 기록을 **직접 입력** (선택 항목) | `@목표 5K 페이스 ≈ …` + "목표 기준" 배지 |
| C. 없음 (현재 기본) | 아무 기록 입력 없음 | 페이스 숫자 없이 `RPE 7~8 · "1500m 경기 느낌"` 감각 앵커만 |

- C단계에서 페이스를 **추정·계산·날조하지 않는다.** 이것은 기존
  `PROFILE_ONLY` 제한 신뢰도 원칙의 연장이다.
- A/B 입력은 계획 인테이크 6번째 화면(선택, 건너뛰기 가능)으로 추가한다.
  건너뛰면 C로 진행 — **입력 강요 없음.**
- 환산표(RP→구간 시간)는 단순 나눗셈만 사용한다(400m @4:10/km = 100초).
  VDOT류 생리학 공식 도입은 이 기획 범위 밖(별도 오너 결정 필요).

## 5. UI 설계 — 3층 카드 (기존 화면에 삽입)

후보 비교(`PlanCandidates`)와 진행(`ActivePlan`)의 세션 행을 카드로 승격한다.

```text
┌──────────────────────────────────────┐
│ DAY 4   [V2]  짧은 VO2 인터벌          │  ← 1층: 시스템 배지(기존 색) + 쉬운 이름
│ 10×400m ×2set @5K RP · r70″ · R10′   │  ← 2층: 숫자 한 줄 (mono, 탭 가능한 토큰)
│ 산소 엔진 출력을 올려요 · 25~40′       │  ← 목적 1문장 + 총 소요
│                            [자세히 ∨] │
├─ 접힌 영역 (탭 시) ────────────────────┤
│ • 풀이: 400미터를 5K 페이스로 10번,     │  ← 3층: 완문 풀이
│   사이 70초 휴식. 10분 쉬고 1세트 더.   │
│ • 이렇게 느껴야 해요: 마지막 2~3개가    │  ← 감각 가이드 (RPE 대체 언어)
│   힘들지만 자세가 무너지진 않는 정도    │
│ • 줄여도 돼요: 컨디션이 별로면          │  ← 하향 조정 허가 (상향 제안 금지)
│   1세트만 하고 마쳐도 훈련은 성립해요   │
└──────────────────────────────────────┘
```

디자인 원칙:
- **1·2층만 봐도 훈련 가능.** 3층은 순수 선택. 접힌 상태가 기본값.
- 조정 안내는 **하향만**("줄여도 돼요"). "더 할 수 있으면 늘리세요"류
  상향 문구는 금지 — 자동 강도 상승 없음 원칙의 카피 버전.
- 배지 색은 기존 6색 그대로. 색맹 대비 배지에 2글자 코드(V2 등) 항상 병기
  (색 단독 의미 전달 금지 — 기존 접근성 원칙).
- 카드 전체 44px+ 터치, mono 표기는 16px 이상(숫자 가독).
- 저모션: 접기/펼치기는 `prefers-reduced-motion` 시 무애니메이션.

## 6. 흥미·집중 유지 장치 (과하지 않게)

- **"오늘의 왜" 1줄**: ActivePlan에서 오늘 세션 카드 상단에만 목적 문장 노출.
  9일치 전체에 설명을 다 펼치지 않는다 — 정보 피로 방지.
- **후보 비교 개선(기존 검수 U3 해소 겸)**: 후보 상단에 시스템 배지 분포
  띠(예: BA×4 · LT×1 · V2×1 · RE×3)를 놓아 두 후보 차이가 한눈에 보이게.
- **완료 후 사실 영수증 연동**: 인터벌 완료 체크 시 "V2 세션 1회 완료가
  저장되었습니다" — 기존 영수증 문법 유지, 칭찬·점수 인플레 없음.
- 하지 않을 것: 뱃지 남발, 애니메이션 축하, "이 훈련을 놓치면" 류 손실 문구.

## 7. 단계별 도입 (각 단계가 독립 배포 가능)

| 단계 | 내용 | 위험 |
|---|---|---|
| P1 | 세션 라벨·목적 1문장·시스템 배지 (카피+표시만, 엔진 무변경) | 낮음 |
| P2 | 숫자 표기 + 토큰 TermHelp + 3층 카드 + 후보 배지 띠 | 낮음 |
| P3 | 인테이크 6화면(기록/목표 입력, 선택) + RP 환산 (A/B단계) | 중간 — 별도 검수 |
| P4 | 경험 밴드별 라이브러리 확장(해당계·복합·대시) | 중간 — 별도 검수 |

P1·P2는 처방 수치를 바꾸지 않는다(표현 계층만). 엔진의 후보 생성 로직 변경은
P4에서 처음 발생하며 그때 impl 계약 테스트를 함께 확장한다.
