## 검토 시작 전 확인서

이 양식은 합성 자료 검토 연습용입니다. 서명해도 실제 참가자를 등록하거나 프로토콜,
런타임, 훈련 계획을 승인하지 않습니다.

| 항목 | 검토자 A | 검토자 B |
|---|---|---|
| 이름·역할 |  |  |
| 검토 날짜·시각 |  |  |
| 패킷 식별자·해시 |  |  |
| 규칙·스키마·안전 버전 |  |  |
| 연결 코치 또는 독립 검토자 구분 |  |  |
| 이해충돌 없음 / 있음과 사유 |  |  |
| 패킷 작성·규칙 작성에 참여하지 않음 |  |  |

검토자 B는 선수, 보호자, 연결 코치, 후보·규칙·패킷 작성자, 운영자 또는 결과에
재정·감독·가족·선발 이해관계가 있으면 독립 검토자가 될 수 없습니다. 두 검토자는
패킷을 보기 전에 이해충돌을 밝힙니다. 충돌이 있으면 판정은 독립 증거가 아닙니다.

## 전수 검토 기록

이 제안에서는 표본을 뽑지 않고 기준 자료 1회와 세 프레임을 합친
**4/4 체크포인트 전수 검토**를 합니다. 합성 중요·공유·검토자 시나리오도 모두 확인하며,
**유리한 사례만 골라 검토하지 않습니다.**

| 확인 항목 | A 기록 | B 기록 |
|---|---|---|
| 같은 가림 처리 패킷과 해시인가 |  |  |
| 원천 자료를 쓸 수 있는가 |  |  |
| 정확한 버전이 고정됐는가 |  |  |
| 비공개 메모 신호가 없는가 |  |  |
| 실제 계획·달력 변경이 없는가 |  |  |
| 결과 또는 결과 없음이 유형으로 기록됐는가 |  |  |
| 안전 경로를 그대로 보존했는가 |  |  |
| 추가 설명이 필요한가 |  |  |
| 판정과 근거 코드 |  |  |
| 잠금·서명 시각 |  |  |

A와 B는 **서로의 판정을 보기 전에 잠금**과 서명을 마칩니다. 검토 기록에는 역할,
이해충돌 선언, 패킷 해시, 버전, 판정, 시각만 남깁니다. 원문 메모나 선수가 말한
상세 이유는 넣지 않습니다. 검토 피드백은 자료 정정 제안까지만 할 수 있고, 실제
계획 변경·일시 정지 해제·안전 승인에는 쓸 수 없습니다.

## 불일치와 조정 기록

A와 B의 판정이 다르면 평균을 내거나 더 좋은 판정을 고르지 않습니다. **불일치**는
그대로 잠그고, 사전에 지정된 **제3 독립 조정자**가 있을 때만 별도 결정을 덧붙입니다.

| 조정 확인 | 기록 |
|---|---|
| A·B 잠금 판정과 시각 |  |
| 조정자 C 이름·역할 |  |
| C가 A·B·선수·코치와 독립적인가 |  |
| C가 후보·규칙·패킷·기존 판정을 작성하지 않았는가 |  |
| C 이해충돌 선언 |  |
| 조정 결과와 허용된 근거 코드 |  |
| 추가 기록 시각과 참조 |  |

C가 없거나 충돌이 있으면 결과는 `UNRESOLVED`로 남고 완료를 멈춥니다. 책임자나 연결
코치가 몰래 동점 결정을 대신하지 않습니다. 기준 버전이 중간에 바뀌면 기존 판정을
이어 붙이지 않고 검토를 다시 시작합니다.

## 파일럿 종료 뒤 기록과 시나리오 추적

**파일럿 종료 뒤**에는 새 처리를 멈추고 다음 중 하나만 제안합니다.
`프로토콜 폐기`, `합성 연습 수정 후 반복`, `별도 파일럿 결정 검토 대상`.
어느 것도 런타임이나 실제 계획을 켜지 않습니다. 이해도와 구분 가능성은 서술형으로
보고하며, **안전성·효과·정확성이 입증되었다는 뜻이 아닙니다.**

| ID | 검토 양식이 강제하는 처리 |
|---|---|
| HR-01 | 두 검토자가 서로 보지 않고 잠금 |
| HR-02 | 패킷 해시 불일치 시 판정 거부 |
| HR-03 | 이해충돌이 있으면 독립 판정 아님 |
| HR-04 | 불일치 때 독립 조정 기록 추가 |
| HR-05 | 조정자 없으면 미해결로 완료 중단 |
| HR-06 | 기준 버전 변경 시 검토 재시작 |
| HR-07 | 검토 의견으로 실제 계획을 바꾸지 않음 |
| HR-08 | 원문 메모·이유가 기록되면 거부 |
| HR-09 | 기존 작성·충돌한 조정자는 무효 |
