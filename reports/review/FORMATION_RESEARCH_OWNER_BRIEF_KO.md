# 9.5일 Formation 연구 책임자 브리핑

## 지금 확정된 것

- 최신 책임자 결정이 이전 draft·acceptance·deferred 문서보다 우선한다.
- TRAINORACLE의 정체성은 **9.5일 기본 자동 처방**이다.
- 9.5일 안에 MAIN 2-3회, 경기 앵커, 지원·회복 블록을 구조화한다.
- 적격 후보가 없으면 다른 주기를 늘어놓지 않고 코치 작성 계획으로 넘긴다.
- 현재는 연구·안전·개인정보·책임자 게이트가 남아 있어 런타임을 켜지 않는다.

## 연구가 지지하는 것

- 일부 선수·코치는 실제 9일·10일 주기를 사용했다. 흔하다는 근거는 없다.
- 고정 7일은 일정·회복·세션 충돌을 만들 수 있지만 직접 우월성 시험은 없다.
- 회복 반응은 24-96시간에 지표·세션·선수별로 달라 72시간은 허가가 아니다.
- 복합 세션은 구성요소·순서·간격을 보존해야 한다.
- 부하는 내적·외적·파생·맥락 값과 단위·출처를 분리해야 한다.
- 개인 분석은 관찰부터 사전 비교 가설까지 단계적으로 제한해야 한다.
- 책임자 미승인 검토 후보로, 달력 대회 1개 아래 예선·결승·복수 종목의 실제 출발을
  각각 보존하는 안이 생겼다. CA-02/03 결정 전에는 현재 counting을 바꾸지 않는다.
- 청소년 우려 입력은 자동 진단이 아니라 후보 중단과 로컬 지원 선택지에만 쓴다.
  답변 거부·미응답은 아무에게도 자동 전송하지 않고, 실제 인계는 선수가 수신자와 범위를
  확인해 따로 선택한다.

## 연구가 지지하지 않는 것

- 정확한 9.5일이 과학적으로 최적·우월·안전하다는 주장.
- 72시간이면 회복·준비·다음 MAIN 허가라는 주장.
- 보편 복합훈련 순서·피로점수·ACWR 안전구간.
- 모든 지표에 같은 최소 기록 수·신선도.
- 코치와 비슷한 계획이 나오면 효능·안전이 입증됐다는 주장.

## 사용자 관점의 중요한 수정

1. 기존 `코치만 최종 선택` 스펙을 `기본안 1개 자동 선택 + 청소년 실제 실행 확인`으로
   고쳐야 한다.
2. 개인 메모는 분석에서 완전 무신호지만, 사용자가 원하면 로컬 전체 백업과 선택 공유를
   할 수 있도록 Daily Log의 절대 금지를 고쳐야 한다.
3. 선수 첫 화면에는 기본 계획 1개·오늘 할 일·대회·확인 필요 사항만 먼저 보여준다.
4. 9.5일 마지막 반나절과 실제 날짜 경계를 모바일에서 명확히 보여준다.
5. 자동 계획이 멈추면 현재 계획과 지금 할 수 있는 행동·해결자를 보여준다.
6. 중학생·코치·보호자·보조기술 teach-back 계획은 준비됐지만 실제 검토는 0건이다.

스포츠과학, N-of-1 통계, 청소년 스포츠의학/safeguarding, 개인정보, 실제 선수·코치와
보조기술 접근성 검토가 남아 있다. AI 선별·추출·평가는 167개를 모두 준비했지만 사람
확인은 0개다. 새 보충 후보 18개도 정본에 합치지 않고 사람 선별을 기다린다. 자동 처방
런타임은 계속 꺼져 있다.

보충 후보 18개의 PMID·DOI·제목은 PubMed ESummary로 다시 대조했고, 5개는 정본과의
중복 후보로 분리했다. 이것은 식별자 확인일 뿐 인간 선별이나 과학적 채택은 아니다.
