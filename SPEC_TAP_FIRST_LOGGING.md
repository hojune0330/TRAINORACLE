## 1. 전 표면 터치 감사 (현행 실측 — LogEntry.tsx 기준)

"키보드가 뜨는 순간 = 마찰"로 정의하고 입력 필드 전수 분류:

### 훈련 후 (post-session)
| 필드 | 현행 | 판정 | 처방 |
|---|---|---|---|
| 강도 시스템 | 칩 6개 탭 | ✅ 이미 터치 | 유지 |
| 세션 제목 | **text input** | ❌ 키보드 | 시스템 선택 시 자동 제목("BASE 러닝") + 수정만 선택적 |
| 거리 | **text input** | ❌ 키보드 | **프리셋 칩** 3·5·8·10·12km + −/＋ 0.5 스테퍼 |
| 시간 | **text input** | ❌ 키보드 | 프리셋 칩 20·30·40·60·90min + −/＋ 5 스테퍼 |
| 평균 페이스 | **text input** | ❌ 키보드 | 거리·시간에서 **자동 계산** 표시 (입력 자체 제거) |
| RPE | 10칸 그리드 탭 | ✅ | 유지 |
| 메모 | textarea | ⚠️ 자유 입력 | **선택으로 강등** — 빈 채 저장 OK + 기분 이모지 대체 가능 |
| 사진/음성 | 데모 표면 | — | 범위 외 (미디어 계약 미수용) |

### 하루 마무리 (evening)
| 필드 | 현행 | 판정 | 처방 |
|---|---|---|---|
| 수면 시간 | range 슬라이더 | ⚠️ 드래그 | 0.5h 칩(6/6.5/7/7.5/8/9) + 스테퍼 병행 |
| 수면 질 / 감정 | 5칸 탭 | ✅ | 유지 |
| 체중·심박 | **text input** | ❌ 키보드 | **직전 값 ±0.1/±1 스테퍼** + "어제와 같음" 원탭. 미측정 = 그냥 스킵 |
| 통증 다이어그램 | SVG 탭 (1→5→해제) | ✅ | 유지 (안전 배너 불변) |
| 오늘의 한 줄 | textarea | ⚠️ | 선택으로 강등 |

### 경기 (race)
| 필드 | 현행 | 판정 | 처방 |
|---|---|---|---|
| 직전: 긴장도·컨디션 | 탭 그리드 | ✅ (단, 현재 **state 미연결 데모**) | state 연결 필수 |
| 직전: 목표 페이스·혼잣말 | text | ❌ | 선택으로 강등 |
| 직후: 기록 | **text input** | ❌ | **분:초.센티 3열 휠/스테퍼** (숫자만, 키보드 없이) — 유일하게 정밀값 필요 |
| 직후: 순위 | text | ❌ | 칩: 1위·2위·3위·입상·완주 + 직접 입력(선택) |

### 기타 표면
| 표면 | 판정 |
|---|---|
| 진입점 선택(chooser) | ✅ 이미 3탭. **시간대 스마트 정렬** 추가 (저녁이면 '하루 마무리' 첫 번째) |
| athletetime /records | ✅ 트랙 J(단계형 선택 위저드)가 이미 동일 철학 — 발주 완료 상태 |

## 2. 핵심 설계 — 퀵 모드(기본) + 자세히 모드(선택)

### 원칙
1. **기본 진입 = 퀵 모드**: 한 화면 한 질문, 큰 터치 타깃, **선택 즉시 자동 다음 화면**(토스 스텝). 텍스트 필드 0개.
2. 모든 화면에 `자세히 쓰기` 링크 → 현행 풀 폼(디테일 모드)으로 전환. 퀵에서 고른 값은 그대로 승계.
3. **터치 예산(수용 기준)**: 훈련 후 ≤ 5탭 · 하루 마무리 ≤ 7탭 · 경기 직전 ≤ 4탭. 예산 초과 설계는 반려.
4. **"지난 훈련 그대로" 원탭 프리필**: 퀵 1화면 최상단에 직전 세션 카드("어제: BASE 8km 40min · RPE 5") — 탭 1번이면 RPE 질문으로 직행.

### 퀵 플로우 — 훈련 후 (최소 4탭)
```
[탭1] 새 일지 → 훈련 후
[화면 1] 뭘 뛰었어요?
   ┌ 지난 훈련 그대로 (BASE 8km 40min) ┐  ← 탭2 → 화면3 직행
   └ 시스템 칩 6개 (탭2) → 화면2
[화면 2] 얼마나요?  거리 칩 + 시간 칩 (탭3·탭4, 페이스 자동)
[화면 3] 얼마나 힘들었어요?  RPE 10칸 (탭5 → 자동 다음)
[화면 4] 저장 완료 ✓ — "한 줄 남길래요?" (선택: 손글씨 입력 or '오늘은 패스')
```
저장은 화면 3 RPE 탭 시점에 즉시 완료(낙관 저장). 화면 4의 메모는 있으면 append.

### 퀵 플로우 — 하루 마무리 (최대 7탭)
```
수면 칩(탭1) → 수면 질(탭2) → 감정(탭3) → 통증? [없음(탭4·끝) | 있음 → 다이어그램 탭들] → 완료
체중·심박은 퀵에서 아예 안 물음(디테일 전용) — "안 잰 날이 더 많다"가 현실.
```

### 퀵 플로우 — 경기 직전 (3탭)
```
긴장도(탭1) → 컨디션(탭2) → 완료(탭3). 혼잣말은 완료 화면 선택 항목.
```

### 완료 화면 공통
- 큰 체크 + "오늘도 1페이지 쌓였어요" + 이번 주 작성일 도트(7칸).
- **금지**: 연속 기록 스트릭 압박·"내일도 꼭!" 류 부추김 문구. 도트는 사실 표시만.

## 3. 불변식 (안전·계약)

1. **JournalEntry 스키마 불변** — 퀵 모드는 같은 `saveEntry()` 경로. 빈 문자열 필드 허용은 기존과 동일.
2. **통증 4~5 REVIEW 배너**: 퀵 통증 화면에도 동일 노출. 안전 판정 해제 없음.
3. **memo_policy §8**: 자유 입력은 여전히 기기 전용 + PrivacyNote. 퀵에서 메모를 "선택"으로 강등하는 것이지 정책 변경 아님.
4. 훈련량 부추김 금지: 프리셋 칩은 직전 값 중심 정렬(큰 값 유도 금지), 완료 화면 문구 심사 대상.
5. `?uitest` 콘솔 마커: `[JSAVE] kind=* ok=*` 유지 + 신규 `[QUICKLOG] step=* taps=*`.
