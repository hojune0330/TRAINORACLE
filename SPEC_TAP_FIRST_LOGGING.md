## 1. 전 표면 터치 감사 (현행 실측 — LogEntry.tsx 기준)

"키보드가 뜨는 순간 = 마찰"로 정의하고 입력 필드 전수 분류:

### 훈련 후 (post-session)
| 필드 | 현행 | 판정 | 처방 |
|---|---|---|---|
| 강도 시스템 | 칩 6개 탭 | ✅ 이미 터치 | 유지 |
| 세션 제목 | **text input** | ❌ 키보드 | 시스템 선택 시 자동 제목("BASE 러닝") + 수정만 선택적 |

| 평균 페이스 | **text input** | ❌ 키보드 | 거리·시간에서 **자동 계산** 표시 (입력 자체 제거) |

## 2. 핵심 설계 v2 — "잉크 스택" 퀵 로그 (한 화면 한 맥락 + 쌓임의 시각화)

> v1(한 화면 한 질문 연쇄)은 페르소나 리뷰에서 반려됨 —
> 결함 D1 화면 쪼개기 과잉 / D2 답의 행방불명 / D3 축적 피드백 부재.
> (근거: reports/review/TAP_FIRST_V1_PERSONA_REVIEW.md)

### 2.1 원칙 (v2)
1. **한 화면 한 맥락**: 질문 단위가 아니라 맥락 단위로 화면을 나눈다.
   같은 맥락의 값(거리+시간, 수면 시간+질)은 **한 화면에 병렬 배치** —
   "40분에 8km"는 한 몸의 정보라서 같이 보여야 그 자리에서 페이스 감이 온다.
2. **전환 예산 신설**: 터치 예산에 더해 **최대 화면 전환 수**를 계약한다.
   훈련 후 ≤ 2전환 · 하루 마무리 ≤ 2전환 · 경기 직전 ≤ 1전환.
   (v1은 3~4전환 — 정보 밀도 낮은 화면의 연쇄가 체감 시간을 지배했다)
3. **터치 예산 유지**: 훈련 후 ≤ 5탭(프리필 경로 ≤ 3탭) · 하루 마무리 ≤ 7탭(통증 없음 4탭) · 경기 직전 ≤ 4탭.
4. 텍스트 필드 0개(퀵) · `자세히 쓰기`로 풀 폼 전환(값 승계) · "지난 훈련 그대로" 원탭 프리필 — v1 유지.

### 2.2 잉크 스택 캔버스 — 이 제품의 고유 틀 ★
토스의 "쌓임의 시각화"를 이 제품의 종이·손글씨 정체성으로 번역한 핵심 장치.

- 화면 상단 40%에 **오늘의 일지 종이**(paper-grid 미니 카드)가 상주한다.
- 아래 60%의 칩/그리드에서 값을 탭하면, 그 값이 **종이에 한 줄 잉크로 적힌다**
  (fade-in + 미세 상승 모션, 손글씨/mono 혼용 — LogDetail의 최종 렌더와 같은 문법).
- **답의 행방불명 해결**: 지금까지 고른 모든 값이 종이 위에 계속 보인다.
  종이의 줄을 탭하면 해당 입력이 다시 열려 **그 자리에서 수정** — 뒤로가기 없음.
- **파생값의 마법 순간**: 거리와 시간이 모두 적히면 페이스가 잉크색 강조로
  **스스로 계산되어 적힌다** (`≈ 5'00"/km`) — "반영되고 있다"를 가장 강하게 인식시키는 순간.
- 화면이 전환될 때 종이는 **그대로 남고 아래 입력부만 교체**된다 —
  "넘어가도 내 답은 쌓여 있다"가 구조적으로 보장됨.
- 저장 = 종이에 **날짜 스탬프가 쾅 찍히는 모션**(Stamp 프리미티브) 후 완료 화면.

### 2.3 퀵 플로우 v2

**훈련 후 — 화면 2장 + 완료** (전환 2회, 3~5탭)
```
[화면 1 · 훈련]  종이(빈 페이지, 오늘 날짜만)
  ├ 지난 훈련 그대로 카드 (탭 → 종이에 3줄이 촤르륵 적힘 → 화면 2로)
  └ 시스템 칩 6개 + 거리 칩 + 시간 칩 (같은 화면, 탭마다 종이에 1줄씩,
     거리·시간 완성 시 페이스 자동 기입)  → 3값 채워지면 자동 전환
[화면 2 · 몸]   종이(위 값 유지) + RPE 10칸
  └ RPE 탭 → 종이에 적힘 → 스탬프 쾅(낙관 저장) → 완료 화면
[완료]  종이 전체가 오늘의 일지로 완성된 모습 + "한 줄 남길래요? / 오늘은 패스"
```

**하루 마무리 — 화면 2장 + 완료** (전환 2회, 4~7탭)
```
[화면 1 · 몸 상태]  수면 칩 + 수면 질 5칸 + 감정 5칸 (한 화면 — 모두 같은 '오늘 몸' 맥락)
  └ 3값 채워지면 자동 전환. 각 탭마다 종이에 적힘.
[화면 2 · 통증]  "통증 없음" 대형 버튼 | 다이어그램 탭들 (REVIEW 배너 동일)
[완료]  체중·심박은 퀵에서 안 물음(디테일 전용).
```

**경기 직전 — 화면 1장** (전환 1회, 2~3탭)
```
긴장도 10칸 + 컨디션 5칸 (한 화면) → 완료. 혼잣말은 완료 화면 선택.
```

### 2.4 마이크로인터랙션 계약 ("느낌 예산") ★
토스가 빠르게 *느껴지는* 이유는 화면 수가 아니라 매 단계의 반응이다. 다음을 스펙으로 계약한다:

| # | 순간 | 반응 | 시간 |
|---|---|---|---|
| M1 | 칩 탭 | 눌림(scale 0.97) + 즉시 선택색 | ≤ 100ms |
| M2 | 값이 종이에 적힘 | fade-in + 2px 상승, 잉크색 | 200ms |
| M3 | 파생값(페이스) 등장 | 0.3s 지연 후 등장 — "계산해 줬다"는 인지 | 300ms |
| M4 | 화면 전환 | 입력부만 좌로 슬라이드, 종이는 고정 | 250ms |
| M5 | 저장 스탬프 | 스탬프 scale 1.4→1.0 쾅 + 종이 미세 흔들림 | 400ms |
| M6 | 완료 화면 | 주간 도트 중 오늘 칸이 마지막에 채워지는 모션 | 300ms |

- 전부 CSS transition/animation으로 구현 가능해야 함 (외부 애니메이션 라이브러리 금지).
- `prefers-reduced-motion` 존중: 모션 제거하고 상태 변화만 즉시 반영.
- 모든 모션은 **정보를 나르는 경우에만** 허용 (장식성 금지 — Tufte 원칙).

### 2.5 완료 화면 공통
- 완성된 오늘의 종이(스탬프 포함) + 이번 주 작성일 도트 7칸(오늘 칸이 채워지는 모션).
- **금지**: 연속 기록 스트릭 압박·"내일도 꼭!" 류 부추김 문구. 도트는 사실 표시만.

## 3. 불변식 (안전·계약)

1. **JournalEntry 스키마 불변** — 퀵 모드는 같은 `saveEntry()` 경로. 빈 문자열 필드 허용은 기존과 동일.
2. **통증 4~5 REVIEW 배너**: 퀵 통증 화면에도 동일 노출. 안전 판정 해제 없음.
3. **memo_policy §8**: 자유 입력은 여전히 기기 전용 + PrivacyNote. 퀵에서 메모를 "선택"으로 강등하는 것이지 정책 변경 아님.
4. 훈련량 부추김 금지: 프리셋 칩은 직전 값 중심 정렬(큰 값 유도 금지), 완료 화면 문구 심사 대상.
5. `?uitest` 콘솔 마커: `[JSAVE] kind=* ok=*` 유지 + 신규 `[QUICKLOG] step=* taps=* screens=*`.
6. **전환 예산(§2.1-2)과 잉크 스택 불변**: 화면 전환 시 종이(누적 답)가 사라지는 설계 금지.
   모든 파생값(페이스)은 잉크 스택에서 자동 기입 — 사용자 입력 표면 금지.
