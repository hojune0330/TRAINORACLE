# 06. 검증 & 세이프가드 (Validation & Safeguards)

> 본 문서는 **시스템 전체의 안전 장치**를 정의한다.  
> 개별 검증 항목 LEGACY_PHASE_D.D-1 ~ LEGACY_PHASE_D.D-9의 상세 규칙은 `05b_RULE_VALIDATOR.md` 참조.

## 1. 데이터 정확성 세이프가드 (Tier 1 — 절대 금지)

> 출처: `진도군청_육상팀_에이전트_절대_금지_사항.md` 정본.  
> **이 5개 룰은 절대 위반 불가**. 위반 시 작업 중단.
> 본 문서는 GLOSSARY.md §0.5 Canonical Safety Statements (S-01 ~ S-04)를 따릅니다. 충돌 시 §0.5가 우선합니다.


| ID | 규칙 |
|---|---|
| **SG-1** | 이전 연도 데이터를 다른 연도에 그대로 적용하지 않는다 |
| **SG-2** | 확인되지 않은 데이터를 절대 생성하지 않는다 |
| **SG-3** | 요약표만 보고 판단하지 않는다 (개별 보고서 교차 검증 필수) |
| **SG-4** | "아마도", "대략", "예상", "비슷할 것" 등 추측성 표현 금지 |
| **SG-5** | 패턴/경향으로 데이터 채우지 않는다 |

**위반 시 처리**:
1. 즉시 작업 중단.
2. "X 선수의 Y 데이터가 확인되지 않습니다" 보고.
3. 추측 없이 코치 입력 대기.

## 3. 훈련 검증 종합 판정 (Tier 3 — Phase D)

> 출처: `phase_validate_v1.3.1.md` §5, §6.  
> LEGACY_PHASE_D.D-1 ~ LEGACY_PHASE_D.D-9 9개 훈련검증 항목의 결합 룰.

### 3.1 등급 정의

| 등급 | 의미 | 즉시 조치 |
|---|---|---|
| **OK** | 준수 | 없음 |
| **WARN** | 주의. 악화 가능성 | 모니터링 강화 |
| **ERROR** | 즉시 대응. 부상·과훈련 위험 | 부하 조정·디로딩 |

### 3.2 종합 판정 규칙

| 조건 | 판정 |
|---|---|
| ERROR ≥ 1건 | **즉시 대응 권고** |
| 서로 다른 LEGACY_PHASE_D.D-1~LEGACY_PHASE_D.D-9에서 WARN ≥ 3건 AND ERROR 0건 | **주의 관찰 권고** |
| WARN ≤ 2건 | **양호** |

> "서로 다른" 항목 카운트가 핵심. 같은 LEGACY_PHASE_D.D-1에서 WARN 5건 발생해도 "1건"으로 센다.

### 3.3 디로딩 자동 트리거

| 트리거 | 조건 | 조치 |
|---|---|---|
| **DT-1** | LEGACY_PHASE_D.D-8 ERROR (3회 연속 페이스 미달) | 디로딩 주 삽입 권고 |
| **DT-2** | LEGACY_PHASE_D.D-9 ERROR (부상 위험) | 즉시 부하 감소 권고, 해당 부위 HIGH 제외 |

디로딩 주 규칙 (출처: `phase_execute_v1.3.1.md` §3 E-D1):
- 볼륨 0.5–0.6배.
- MAIN 0–1회.
- Z1 비중 30 % 이상.
- 복귀 시 디로딩 전 80 %에서 시작, 주당 +10 % 이하.

## 4. 충돌 해결 우선순위 (Tier 4 — 다중 경고 동시 발생)

> 출처: `phase_execute_v1.3.1.md` §3 "충돌 해결 순서".  
> 위쪽이 우선. 상위 제약 먼저 적용, 하위는 상위 범위 내에서만.
> Tier 0 notice: 현재 SPEC 레이어의 `RULE_SPEC_D1_D9.D-9` safety hard-stop은 본 legacy Phase E 충돌 해결표보다 우선한다. 코치 명시 지시는 legacy 훈련 설계 범위 안에서만 1순위이며, safety hard-stop을 해제하거나 우회할 수 없다.

| 순위 | 항목 |
|---|---|
| 1 | 코치 명시 지시 |
| 2 | LEGACY_PHASE_D.D-9 (부상) |
| 3 | LEGACY_PHASE_D.D-8 (디로딩) |
| 4 | LEGACY_PHASE_D.D-6 (CK) |
| 5 | LEGACY_PHASE_D.D-2 (간격) |
| 6 | LEGACY_PHASE_D.D-3 (Z1) |
| 7 | LEGACY_PHASE_D.D-5 (GLY/VO2) |
| 8 | LEGACY_PHASE_D.D-1 (비중) |

**불가능 시 처리**: 다음 마이크로사이클로 이월, 보고서에 명시.

**예시**: LEGACY_PHASE_D.D-6 ERROR(간격 4일 강제) + LEGACY_PHASE_D.D-1(GLY-SHORT 추가 필요) 동시 발생  
→ LEGACY_PHASE_D.D-6이 4순위, LEGACY_PHASE_D.D-1이 8순위 → LEGACY_PHASE_D.D-6 우선 → 간격 4일 유지, 빈 슬롯에 GLY-SHORT 시도 → 없으면 "다음 마이크로사이클로 이월" 기록.

> Safety override notice: 코치 명시 지시는 훈련 설계 우선순위에서는 중요하지만,  
> `RULE_SPEC_D1_D9.D-9` safety hard-stop을 해제하거나 우회할 수 없다.

## 5. 불확실성 마커 (Markers)

> 출처: `C_workflow_engine_v1.3.md` 규칙 4.

| 마커 | 의미 | 사용 |
|---|---|---|
| `[추정]` | 데이터 부족으로 추정한 값 | 페이스 미제공, Level 1 데이터 등 |
| `[?]` | 분류·판정 신뢰도 낮음 | MOD-HIGH 경계, GLY/VO2 겹침 영역, Z1 90–100″ 경계 |
| `[수정됨]` | 코치가 수동 수정 | Phase C 코치 피드백 후 |
| `[재분류]` | 보조 → 메인 재분류 | HIGH 이상으로 판정된 보조 세션 |
| `[부상 위험 주의]` | LEGACY_PHASE_D.D-9 WARN | 해당 부위 모니터링 |
| `[부상 위험 경고]` | LEGACY_PHASE_D.D-9 ERROR | CMD-DIAGNOSE 권고 |

## 6. 자가 점검 체크리스트 (Tier 5 — 운영 매니저 일일 체크)

> 출처: `진도군청_육상팀_에이전트_절대_금지_사항.md` §📋

### 6.1 데이터 정리 작업 전

- [ ] 해당 연도의 원본 자료를 모두 확보했는가?
- [ ] 각 선수별로 개별 문서를 확인했는가?
- [ ] 요약표와 개별 자료가 일치하는가?
- [ ] 추측이나 가정을 한 부분이 있는가?
- [ ] 이전 데이터를 복사한 부분이 있는가?
- [ ] 모든 숫자의 출처를 명시할 수 있는가?
- [ ] 불확실한 부분을 사용자에게 확인 요청했는가?

### 6.2 Phase D 출력 검증

- [ ] LEGACY_PHASE_D.D-1 ~ LEGACY_PHASE_D.D-9 9개 항목이 모두 출력되었는가?
- [ ] 각 항목에 OK/WARN/ERROR/SKIP 명시되었는가?
- [ ] 종합 판정이 §3.2 규칙대로 되었는가?
- [ ] 디로딩 트리거가 §3.3 조건과 일치하는가?

### 6.3 Phase E (DESIGN) 출력 검증

- [ ] 충돌 해결 우선순위(§4)가 적용되었는가?
- [ ] 옵션 2~3개가 제시되고 코치 선택 대기하는가?
- [ ] 생성된 계획에 LEGACY_PHASE_D.D-1~LEGACY_PHASE_D.D-7 자동 재검증을 수행했는가?
