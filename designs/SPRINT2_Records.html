4 타입 기록의 의미

PB · SB는 정체성. Target은 매일 돌아오는 동기. Reference는 비교 좌표. 페이스 환산표는 모든 추천의 기반입니다.

  • PB Trail — 3년 진척 곡선 + 다음 target line. 현재가 어디에 있는지 시각으로.
  • Target gap bar — 매일 들어와 진척 % 변화 확인. 동기 부여 메커니즘.
  • Reference — 라이벌 입력은 사용자가, cohort는 opt-in. 무리한 비교 X.
  • 페이스 환산표 — PB 입력 → 7개 zone 자동 계산. 모든 세션 추천의 기반.
  • 모든 기능 현재 무료. cohort 매칭, 페이스 시뮬레이션, 라이벌 추적도 모두 사용 가능.
  • 추후 Pro 분리 결정 시 변경 쉽도록 코드 구조만 준비. design-system/FEATURE_TIERS.md 참조.
  • 가드레일: narrative 사실만 ("−52초 단축"), 3년 데이터로 cold start 자동 해결, Reference cohort opt-in.
