# Variants — 왜 화면마다 변형이 여러 개인가

> Home 3개 · LogDetail 2개 · Trends 2개. Tweaks 패널에서 전환.
> 각 변형은 **다른 사용자 시나리오** 를 노립니다. "디자이너가 못 정해서 여러 개" 가 아닙니다.

---

## Home — 3 변형

선수가 앱을 열었을 때 **무엇을 가장 먼저 보여줄까** 의 차이.

### A. Journal (디폴트)
- 인덱스 카드 헤더 (날짜·요일·날씨·사이클 라벨)
- 어제 일지의 손글씨 메모 prevew
- **오늘 일지 쓰기 CTA** (가장 큰 시각 무게)
- 주간 거리·강도 분포

**누구를 위해:** 매일 쓰는 게 습관이 된 선수. 일지가 일상의 일부.
**언제 보여주나:** 디폴트. 가장 v3 의 정신에 충실.

### B. Sparkline
- 상단에 작은 sparkline (지난 14일 거리·HR·수면)
- 그 아래 오늘 CTA
- 데이터 중심 — 추세를 한 눈에

**누구를 위해:** "내가 지금 어떤 사이클에 있나" 를 매일 확인하고 싶은 선수. 데이터 친화적.
**언제 보여주나:** 시즌 후반 (경기 다가올수록 컨디션 흐름이 중요).

### C. Recall
- "1년 전 오늘은…" — 작년 같은 날의 훈련/메모 리콜
- 그 아래 오늘 CTA

**누구를 위해:** 일지를 1년 이상 쌓은 선수. **본인 데이터의 가치를 느끼는 순간**.
**언제 보여주나:** 1년차 이후. 신규 사용자에겐 의미 없음.

> **출시 전략:** A 가 디폴트. B 와 C 는 Tweaks 에서 켤 수 있게.
> 사용 데이터 쌓이면 자동 전환 룰 만들기 (예: "1년 이상 사용자 = C 자동 노출").

## LogDetail — 2 변형

과거 일지 한 건을 열었을 때 **무엇을 강조할까** 의 차이.

### A. Journal page (디폴트)
- 인덱스 카드 헤더
- 마일스톤 (PB / Cycle peak / Race day) 이 잉크 스탬프로
- 자동 메트릭 (avg HR · pace · splits) 는 **메모와 같은 페이지에** 평등하게
- 손글씨 메모는 `.paper-lines` 배경 위에
- **인쇄 가능** (`@media print` A5 portrait)

**누구를 위해:** "오늘은 어떤 날이었는가" 를 회상하고 싶은 선수. 정서적 의미가 데이터만큼 중요.
**언제 보여주나:** 디폴트. v3 의 출시 정신.

### B. Dashboard
- 메트릭 카드들이 그리드로 위쪽
- 메모는 아래 별도 섹션
- 데이터가 시각적으로 우선

**누구를 위해:** 코치 모드 / 분석 모드. 한 훈련의 데이터를 빠르게 확인하고 싶을 때.
**언제 보여주나:** "분석 모드" 토글 켰을 때. 또는 코치 페르소나가 선수 일지를 검토할 때 (미래 모듈).

> **출시 전략:** A 가 디폴트. B 는 "데이터 보기" 보조 토글로.

## Trends — 2 변형

지난 N일/주 의 추세를 보여줄 때 **어떻게 묶을까** 의 차이.

### A. Scroll (디폴트)
- 한 페이지에 distance · intensity distribution · sleep · resting HR · RPE 다 세로로 쌓음
- 각 섹션은 작은 카드, sparkline 미니멀
- 손가락으로 쭉 내려가며 본다

**누구를 위해:** "사이클 전체의 그림" 을 한 번에 보고 싶은 사용자. Tufte 적 dense overview.
**언제 보여주나:** 디폴트.

### B. Tabs
- 상단 탭 (Distance · Intensity · Recovery · RPE)
- 각 탭이 그 메트릭에 집중한 큰 차트

**누구를 위해:** 한 메트릭을 깊게 보고 싶을 때. 코치가 선수 데이터 검토할 때.
**언제 보여주나:** 보조. 디테일 분석 시.

> **출시 전략:** A 가 디폴트. B 는 모달이나 큰 화면 (태블릿 / 데스크탑) 에서 우선.

## Tweaks 패널에 노출되는 옵션 (요약)

```
Home variant      [A · Journal] [B · Sparkline] [C · Recall]
Log detail        [A · Journal page] [B · Dashboard]
Trends            [A · Scroll] [B · Tabs]
AI 표시           [ON / OFF]
강도 인코딩       [dot-code / chip / glyph]
```

각 옵션의 디폴트는 **A · ON · dot-code** 입니다. 즉 디폴트 = v3 의 출시 정신.

---

## 출시 후 변형을 줄일 것

변형은 디자이너의 보험입니다. 실 사용자 데이터가 6개월 쌓이면 어떤 변형이 실제로 쓰이는지 보고 **하나만 남기고 나머지를 제거** 할 예정. 변형을 유지하는 건 코드 부담입니다.

**제거 우선순위 (디자이너 추정):**
1. LogDetail B (Dashboard) — 코치 모듈로 분리될 가능성 큼
2. Trends B (Tabs) — 태블릿 전용으로 분리될 가능성 큼
3. Home C (Recall) — 1년 사용자가 충분히 쌓이면 자동화

**절대 안 줄일 것:** Home A · LogDetail A · Trends A 의 정신 — 일지 톤.
