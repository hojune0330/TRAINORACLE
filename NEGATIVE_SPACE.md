# Negative Space — 안 한 것들

> "이 도구는 무엇이 아닌가" 의 목록.
> 새로 합류한 사람이 "이거 추가하면 좋지 않아?" 라고 제안하기 전에 읽어주세요.
> **여기 적힌 건 이미 검토하고 의도적으로 거부한 것** 입니다.

---

## 시각 / 미감

### ❌ 그라데이션 (linear · radial · conic 전부)
- **왜 검토했나:** v1 에서 SaaS 톤으로 쓰던 패턴.
- **왜 거부했나:** 트레인오라클은 측정 도구. 그라데이션은 "느낌" 의 언어이고, 우리는 "수치" 의 언어를 쓴다. 또 인쇄했을 때 토너만 잡아먹고 의미를 안 전달함.
- **예외:** 없음. `.hl` (형광펜) 의 `linear-gradient(transparent 60%, ...)` 는 종이 형광펜 흉내라 예외 아님 — 한 색 60%/40% 분할.

### ❌ 둥근 모서리 (radius > 4px)
- **왜 검토했나:** 모바일 친화적, 부드러운 인상.
- **왜 거부했나:** "연구자 노트북 + 모눈 일지장" 의 정신. 노트북 페이지는 직각이다. 또 인덱스 카드·도서관 카탈로그 미감을 위해 직각 필수.
- **예외:** `border-radius: 50%` 아바타 circle 만. 사람 얼굴은 직사각이 아니므로.

### ❌ 어두운 테마 (Dark mode)
- **왜 검토했나:** v1 이 어두운 테마였음. 또 트레이닝 앱은 야외에서 보는 경우가 많아서.
- **왜 거부했나:** "프린트해서 가지고 싶다" 가 출시 정신. 종이는 흰색 (정확히는 `--paper` 크림색). 어두운 테마는 종이 비유와 충돌. 또 손글씨 미감이 검은 배경에선 죽음.
- **미래 가능성:** "야외 모드" 같은 보조 모드로 검토 가능. 출시판에선 단일 light.

### ❌ SaaS 흔한 일러스트레이션 (3D 캐릭터, 추상 도형 등)
- **왜 검토했나:** 빈 화면 (empty state) 채우려고.
- **왜 거부했나:** 손글씨 일지 미감과 결이 안 맞음. 빈 화면은 **모노 라벨 + 직선** 으로 처리하거나 빈 채로 둠.

### ❌ 이모지를 UI 시스템 영역에서 쓰기
- **왜 검토했나:** 친근함, 톤 부드럽게.
- **왜 거부했나:** 측정 도구의 무게. 호준 코치 표현: "선수들이 이모지 코칭 받으러 오는 게 아니다."
- **예외:** **사용자 입력 영역만** 자유. 메모·일기에 이모지 쓰는 건 사용자 자유.

## 정보 / 인코딩

### ❌ 색만으로 정보 인코딩
- **왜 검토했나:** 강도별로 색 다르게 (BA=초록, V2=빨강 등).
- **왜 거부했나:**
  1. 색맹 사용자가 못 읽음.
  2. **인쇄하면 사라짐** — 출시 정신과 충돌.
  3. 잘 만든 정보 디자인은 색을 보조 채널로만 씀.
- **대신:** 도트 + 코드 (BA · LT · V2 등) + underline 두께. 색은 가장 마지막 채널.

### ❌ AI 가 결정을 대신 내림
- **왜 검토했나:** "이번 주 회복일 추천" 같은 자동화.
- **왜 거부했나:** 호준 코치 표현: "**선수가 본인 몸을 아는 게 우선**. AI 가 '회복일 가져' 라고 하면 선수가 본인을 안 느끼게 됨." AI 는 **패턴을 보여줄 뿐** (예: "최근 3일 RPE 가 높았다") 결정은 사용자.
- **결과:** AI 출력은 1-2 줄, 동사 없이 진술문, 끄기 가능.

### ❌ 게이미피케이션 (배지·streak·레벨)
- **왜 검토했나:** Strava/Nike Run Club 처럼 retention 도구로.
- **왜 거부했나:**
  1. 일지의 정서를 망침. "오늘 streak 끊기면 안 돼" 라는 압박은 부상선수에게 독.
  2. 호준 코치 표현: "선수들은 이미 자기랑 싸우고 있다. 앱이 또 점수 매기지 마라."
- **예외:** **Stamp 컴포넌트 (PB · Cycle peak · Race day)** 는 게이미피케이션이 아니라 **마일스톤 기록**. 사용자가 달성하면 도장 찍힘, 일부러 노리는 게임이 아님.

### ❌ 소셜 (피드·좋아요·팔로우)
- **왜 검토했나:** Strava 의 기본 retention 모델.
- **왜 거부했나:** 일지는 **혼자 보는 것**. 또는 **본인 코치만 보는 것**. 공개 피드는 일지의 솔직함을 죽임.
- **미래 가능성:** "내 코치와 공유" 단방향 공유는 검토 가능. 양방향 소셜은 출시 후에도 안 함.

## UX / 플로우

### ❌ 거대한 단일 일지 폼
- **왜 검토했나:** 모든 필드를 한 화면에.
- **왜 거부했나:** 4 진입점이 다 다른 머리 상태. 훈련 직후 (땀에 절어서) 와 자기 전 (침대에서) 와 경기장 (긴장 상태) 가 같은 폼을 쓸 수 없음.
- **대신:** `LogEntry.jsx` 가 entryType 별로 4 폼 분기.

### ❌ 첫 화면을 AI 챗으로 시작
- **왜 검토했나:** v2 의 정신.
- **왜 거부했나:** AI 가 메인 행위를 끌면 일지가 보조가 됨. v3 는 거꾸로 — 일지가 메인, AI 가 옆.
- **결과:** v2 의 `AIChat.jsx` 는 archive. 추후 코치 전용 뷰로 분리.

### ❌ 온보딩 슬라이드 5장
- **왜 검토했나:** "트레인오라클은 이런 도구입니다…"
- **왜 거부했나:** 일지 도구의 가치는 **써봐야 안다**. 슬라이드로 설명하면 이미 진다. 첫 화면 = 첫 일지 쓰기.
- **대신:** "오늘 일지 쓰기 CTA" 가 첫 화면. 빈 폼 안에 짧은 안내 (`placeholder`) 만.

### ❌ 캘린더를 메인으로
- **왜 검토했나:** v2 의 9.5-cycle 캘린더가 차별화 포인트.
- **왜 거부했나:** 캘린더는 **계획자** 의 도구. 일지는 **현재** 의 도구. 첫 출시는 일지에 집중.
- **미래 가능성:** 캘린더는 v3.5 또는 코치 모듈에서 부활 가능.

## 기술 / 구현

### ❌ 차트 라이브러리 (Chart.js · Recharts 의 기본 스타일)
- **왜 검토했나:** 빠른 차트.
- **왜 거부했나:** 기본 스타일이 우리 미감과 충돌. **Visx 또는 Recharts 를 쓰되 Tufte 스타일 강제** — hairline grid, 단일 색, 면 채움 없음.
- **현재 상태:** Sparkline 은 SVG 직접. Intensity stack 은 CSS flex.

### ❌ shadcn/ui · Material UI 같은 의견 강한 컴포넌트 라이브러리
- **왜 검토했나:** 개발 속도.
- **왜 거부했나:** v3 의 직각·hairline·종이톤이 너무 distinctive 해서 pre-styled 컴포넌트가 모두 충돌. 매번 override 하느니 처음부터 만드는 게 빠름.
- **대신:** Radix UI (headless) + 자체 스타일.

### ❌ 이모지 단축키 (`:fire:` 자동 변환 등)
- **왜 검토했나:** 사용자 입력 편의.
- **왜 거부했나:** 메모는 손글씨 톤. 이모지 자동 변환은 톤 깨는 트리거.

## 비즈니스 / 제품 결정

### ❌ 첫 출시에 코치 페르소나
- **왜 검토했나:** v2 의 Inbox · AIChat.
- **왜 거부했나:** 두 페르소나 (선수 · 코치) 를 동시에 출시하면 정체성이 흐림. **선수용 일지** 가 먼저 단단해진 뒤 코치 모듈 추가.

### ❌ Strava 와 직접 경쟁
- **왜 검토했나:** "AI 가 더 똑똑한 Strava" 포지셔닝.
- **왜 거부했나:** Strava 는 기록·공유 도구. 우리는 **혼자 쓰는 일지**. 경쟁이 아니라 보완. 통합 가능 (Strava 자동 import → 일지에 첨부).

### ❌ AI 코칭 도구로 마케팅
- **왜 검토했나:** 트렌드.
- **왜 거부했나:** 1) 레드오션. 2) 우리 정신과 안 맞음. **"매일 쓰는 손 일지"** 로 포지셔닝.
